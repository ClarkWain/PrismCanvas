从零到一：Polyline2D 算法分析与交互教程

导读： 在现代图形学（如 OpenGL/DirectX/Vulkan）中，原生直接画“带有厚度和样式的线”是一件非常困难的事。通常 GL_LINES 不支持指定的像素厚度，也无法设置拐角样式。因此，我们需要将一条“线”通过数学计算，展开变成多边形（三角形网格）再进行渲染。本项目中引用的 crushedpixel/Polyline2D 便是为了解决这一痛点。

一、为什么我们需要 Polyline2D？

如果你使用过 Photoshop 或 HTML Canvas，你一定觉得画一条粗线很容易，甚至还能设置各种端点（Cap）和连接点（Join）样式。但是在现代 GPU 管线中，硬件只能高效地绘制点（Points）、线段（单像素宽）和三角形（Triangles）。

当我们想用 OpenGL 画一条 10 像素宽的粗线时，唯一正确且兼容性最好的做法是：利用数学算法，以折线骨架的点为基础，生成覆盖这根线的包围多边形，最后将其切割为 GPU 可以识别的三角形网格。 Polyline2D 库的功能正是输入一组 2D 坐标、线宽、连接样式，然后输出排列好的三角形顶点（Vertices）。

二、核心原理：法线向量与顶点展开 (Expansion)

铺垫一下数学基础：我们要怎么把一根没有宽度的线，变成有宽度的长方形？答案是法线向量（Normal Vector）。

  1. 假设有一条线段 $AB$，从 $A(x_1, y_1)$ 画到 $B(x_2, y_2)$。
  2. 算出其方向向量 $\vec{D} = B - A$，并将其标准化为单位向量 $\vec{u}$。
  3. 求出它的垂直向量（法线）：$\vec{n} = (-u_y, u_x)$。
  4. 向两侧沿着法线平移，距离为线宽的一半（$half\_thickness = \frac{W}{2}$）。
  5. 我们就可以得到长方形的四个顶点：$A+\vec{n} \cdot \frac{W}{2}$, $A-\vec{n} \cdot \frac{W}{2}$, $B+\vec{n} \cdot \frac{W}{2}$, $B-\vec{n} \cdot \frac{W}{2}$。

长方形即可拆分为 2 个三角形交由 OpenGL 渲染。

三、拐角的战争：连接样式 (Joint Style)

单条线段很好画，但当多条线段连成折线时，在拐角处，外侧会产生缺口，内侧会产生重叠。在这个时候，如何“补洞”就成了一个非常复杂的问题。Polyline2D 定义了三种不同的 Joint 样式：

  • MITER (斜接)：一直延伸两条相邻侧边，直到它们相交（形成一个尖角）。如果角度太尖导致延伸过长，往往需要退化为 Bevel 以防尖刺。
  • BEVEL (斜面)：直接将两个端点相连，形成一个被削平的直角。这种方式最简单，性能最高。
  • ROUND (圆角)：将断开处画一个圆弧将其顺滑连接。原理是以连接点为圆心，线宽一半为半径构建扇形（Triangle Fan）。

四、代码级映射：Polyline2D 源码剖析

现在回到你的 C++ 项目中，我们来看看 Polyline2D.h 是如何实现上述原理的。这个库被设计为一个典型的泛型（Template）实现。

1. 使用规范

// 从业务逻辑中提供点列
std::vector points = { {10,10}, {50,80}, {100,20} };

// 库负责将其计算为可以交由 OpenGL glDrawArrays 的顶点集合
std::vector vertices = crushedpixel::Polyline2D::create(
    points, 
    20.0f, // 粗细
    crushedpixel::Polyline2D::JointStyle::ROUND,
    crushedpixel::Polyline2D::EndCapStyle::ROUND
);

2. 线段的构建：LineSegment

在库的底层，首先遍历所有输入的 points，相邻两点之间创建一个 LineSegment。

for (size_t i = 0; i + 1 < points.size(); i++) {
    auto &point1 = points[i];
    auto &point2 = points[i + 1];
    
    // 如果两个点不重合，生成片段。这对应于上文原理的第一阶段。
    if (!Vec2Maths::equal(point1, point2)) {
        segments.emplace_back(LineSegment(point1, point2), thickness);
    }
}

3. 圆角的生成：Triangle Fan (顶点扇)

当选中 ROUND 端帽或圆角时，Polyline2D 利用三角函数生成扇形，其主要算法被封装在 createTriangleFan 方法中：

// 截取自 Polyline2D::createTriangleFan
// 根据圆弧角度细分出多小段（segments），形成三角扇
for (int i = 0; i < segments; i++) {
    crushedpixel::Vec2 endPoint = point1;
    // 使用旋转矩阵算出新的点的位置
    endPoint.x = std::cos(rot) * point1.x - std::sin(rot) * point1.y;
    endPoint.y = std::sin(rot) * point1.x + std::cos(rot) * point1.y;
    endPoint = Vec2Maths::add(endPoint, origin);

    // 每次输出一个三角形：中心点 -> startPoint -> endPoint
    *vertices++ = startPoint;
    *vertices++ = endPoint;
    *vertices++ = connectTo; // 也就是圆心

    startPoint = endPoint;
}
注意性能陷阱：
使用 ROUND 虽然视觉极佳，但由于加入了大量的三角函数和切片（Triangles），生成的顶点数远超 MITER 或 BEVEL。针对画笔涂鸦这种可能有成百上千折线的场景，建议谨慎控制扇形的分割粒度（Segments），或者依靠 shader 通过 SDF（Signed Distance Field）做逐像素的圆角抗锯齿，以减轻 CPU 构建批次的压力。

五、如何应用于项目里的渲染器

你的工程采用 OpenGL（opengl/GLProgram.h, opengl/GLVertexArray.h）。一旦通过 Polyline2D 得出 vertices 集合，它本质上就是一长串平面座标：

[ x1, y1,  x2, y2,  x3, y3, ... ] // 三角形 1 的 3 个点
[ x4, y4,  x5, y5,  x6, y6, ... ] // 三角形 2 的 3 个点

此时可以在 RenderContext / DrawLines 中，把这些值直接通过 glBufferData 喂给 OpenGL：

  • 绑定你的 Vertex Array Object (VAO) 和 VBO。
  • 装载这些打平的点数组。
  • 执行 glDrawArrays(GL_TRIANGLES, 0, vertices.size());

这便是现代图形界面框架（如 Dear ImGui, Qt, 或浏览器内核）实现高质量 2D 线条渲染的万恶之源和万用金钥匙！
