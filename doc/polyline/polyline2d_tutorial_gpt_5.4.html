1. Overview

先用人话说清：Polyline2D 到底解决什么问题

先把最重要的一句话记住：Polyline2D 不是在屏幕上“直接画线”，而是在 CPU 上先把线条算成很多三角形，再交给渲染器去画。 这件事听起来拐了一步，但它正好解决了很多渲染 API 的痛点：原生粗线宽往往不稳定、风格不统一、跨平台行为不一致，而三角形是最稳定的通用图元。

你可以把它想成什么 想成一支有粗细的笔，先沿中心线走一遍，再把笔迹的边界变成一片可渲染的面。
它真正负责什么 相邻点变线段、两侧偏移出边界、在拐角和端点补几何、最后吐出顶点列表。
它不负责什么 颜色、纹理、混合、抗锯齿、上传 GPU、真正发 draw call，这些都需要你自己的渲染代码来做。
一句话边界 它解决的是 stroke mesh，也就是“粗描边几何”，不是通用 Path 填充库。
术语 一句话解释 新手此刻先记什么
points 中心线经过的点列。 先把它当成“笔尖要路过的点”。
thickness 总线宽。 内部常会转成 half thickness，也就是半线宽。
JointStyle 折线拐角怎么接。 MITER 尖、BEVEL 平、ROUND 圆。
EndCapStyle 开放折线首尾怎么收尾。 BUTT 不外伸，SQUARE 外伸半线宽，ROUND 补半圆。
allowOverlap 极端拐角下是否允许更激进的求交和重叠。 新手先记成：false 更保守，true 更激进。
Vec2 要求 传给模板的点类型要求很低。 只要有公开 x/y float 字段和 (x, y) 构造函数就够用。 
#include <Polyline2D/Polyline2D.h>

using namespace crushedpixel;

std::vector<Vec2> points = {
    {0.0f, 0.0f},
    {40.0f, 0.0f},
    {70.0f, 30.0f}
};

auto mesh = Polyline2D::create<Vec2>(
    points,
    12.0f,
    Polyline2D::JointStyle::ROUND,
    Polyline2D::EndCapStyle::BUTT,
    false
);

// mesh 是顶点数组，不是像素线。
// 你通常会把它上传到 GPU，然后按三角形去绘制。
无须 GPU 原生粗线支持 支持 join / cap 风格 输出可直接用于 draw triangles
先建立一个关键心智模型： 你不是在调用一个“画线 API”，而是在调用一个“生成线条几何”的库。几何一旦生成出来，渲染器眼里看到的就不再是线，而是一堆普通三角形。
2. Examples

先看 3 个最小例子，再学原理

新手学这类库，最容易一上来就掉进公式里。更好的顺序是先看“输入长什么样，输出会长什么样”。 下面三个例子分别对应三种最常见场景：单段线、带拐角的折线、闭合路径。

例子 C：闭合路径

std::vector<Vec2> pts = {
    {0.0f, 0.0f},
    {60.0f, 0.0f},
    {70.0f, 30.0f},
    {10.0f, 36.0f}

3. Foundation

从一条线段开始：方向、法线、半线宽

现在再进入几何就比较自然了。Polyline2D 的一切都建立在一个非常基础的动作上： 先拿到一段中心线的方向，再求出它的左右法线，把线段向两侧各挪动半个线宽。 如果你能吃透这一步，后面的 join 和 cap 本质上都只是“在边界之间补三角形”。

B - A = (dx, dy) |B - A| = sqrt(dx^2 + dy^2)
dir = normalize(B - A) dir = (dx, dy) / |B - A|
normal = (-dir.y, dir.x) 这表示把方向逆时针旋转 90°
edge1 = center + normal * halfThickness edge2 = center - normal * halfThickness
dir = normalize(B - A) normal = (-dir.y, dir.x) edge1 = center + normal * (thickness / 2) edge2 = center - normal * (thickness / 2)
template<typename Vec2>
struct PolySegment {
    PolySegment(const LineSegment<Vec2>& center, float thickness)
        : center(center),
          edge1(center + Vec2Maths::multiply(center.normal(), thickness)),
          edge2(center - Vec2Maths::multiply(center.normal(), thickness)) {}

    LineSegment<Vec2> center, edge1, edge2;
};
为什么要先求单位方向 因为只有先把长度归一成 1，后面乘半线宽时才不会被原始线段长度污染。
为什么法线是 (-y, x) 这相当于把方向向量逆时针旋转 90°。在这个库里，edge1 可以理解成“左侧边”。
为什么内部常用 half thickness 因为左右各偏一半最自然：线宽 12，就意味着左边 6、右边 6。
为什么跳过重复点 零长度线段没有方向，也没有法线，会造成归一化和交点计算失效。 
// 你可以把最小思路写成这样
Vec2 delta = b - a;
Vec2 dir = normalize(delta);
Vec2 n(-dir.y, dir.x);

Vec2 leftA  = a + n * halfThickness;
Vec2 leftB  = b + n * halfThickness;
Vec2 rightA = a - n * halfThickness;
Vec2 rightB = b - n * halfThickness;

// 这四个点就构成了一段粗线的“壳”。
4. Joints

新手最容易卡住的地方：MITER、BEVEL、ROUND 到底差在哪

直段其实不难，真正决定“这条线看起来像不像你想要的笔触”的，是拐角。 当前一段和后一段相遇时，它们各自都有左右边界，于是你必须回答一个问题：外侧空出来的那一块，拿什么补上？ 这就是 joint style 的职责。

Miter 不是无条件尖角。 如果转角太尖，外侧交点会被拉得非常远，miter 会长成一根很夸张的“刺”。 因此 Polyline2D 在夹角过小时会自动把 MITER 降级成 BEVEL。当前源码阈值约为 20°。
miter 越尖，外侧交点越远 theta 越小，尖刺越长
一个常见近似关系： miterLength ≈ halfThickness / sin(theta / 2)
auto wrappedAngle = angle;
if (wrappedAngle > pi / 2) {
    wrappedAngle = pi - wrappedAngle;
}

if (jointStyle == JointStyle::MITER && wrappedAngle < miterMinAngle) {
    jointStyle = JointStyle::BEVEL;
}
5. Caps

开放折线怎么收尾：BUTT、SQUARE、ROUND，闭合路径为什么用 JOINT

一条开放折线有两个“裸露”的端点。端帽 end cap 决定首尾怎么补几何。 如果折线本来就是闭合的，那么首尾不是“端点”，而是“又一个拐角”，所以闭合情况下应该走 JOINT，而不是普通端帽。

BUTT 平切，不超过真实端点。最朴素，也最“收着”。
SQUARE 仍然平切，但会沿前进方向再外伸半线宽，看起来更饱满。
ROUND 在端点补半圆。视觉最柔和，但也意味着额外三角形更多。
SQUARE cap 的关键动作 pathStart = pathStart - direction * halfThickness pathEnd = pathEnd + direction * halfThickness
auto capStyle = closed
    ? crushedpixel::Polyline2D::EndCapStyle::JOINT
    : toPolylineEndCapStyle(paint.getStrokeCap());

7. allowOverlap

allowOverlap 为什么存在

这是一个很容易被忽略，但很说明实现取舍的参数。它不是“质量开关”，而是“几何取舍开关”： 更保守地避免重叠，还是在极端转角下允许几何相互覆盖，以换更连续的轮廓。

如果你是第一次用它，最实用的记法就是：false 更保守，true 更激进。 当线条有透明度时，重叠区域可能会更深，所以很多人会先从 false 开始；如果你更看重极端转角的连续轮廓，再去试 true。

8. Algorithm

把源码压缩成 5 步：一个新手能抓住的最小算法

如果你现在只想把整个流程装进脑子里，不想一次吞完整个头文件，可以先记这 5 步。 这是一个“够对、够用、够稳定”的理解版本，适合新手建立第一版心智模型。 

// 最小版伪代码
sanitize points

for each adjacent pair:
    build center segment
    compute left/right offset edges

handle start cap / end cap

for each segment:
    if next segment exists:
        build joint between current and next

    emit two triangles for the current strip

return all vertices
两条直线求交常写成： p = a0 + t * r q = b0 + u * s
如果 cross(r, s) ≈ 0 说明两线平行或近似平行
// 交点的一种常见写法
Vec2 r = a.b - a.a;
Vec2 s = b.b - b.a;
Vec2 d = b.a - a.a;

float denom = cross(r, s);
float t = cross(d, s) / denom;

Vec2 intersection = a.a + r * t;
  1. 先清洗输入点，去掉会造成零长度线段的重复点。
  2. 每两点组成一个中心线 segment，再生成左右两条 offset edge。
  3. 对开放路径处理端帽，对闭合路径把首尾再走一次 joint。
  4. 每段直段固定吐两个三角形，拐角和圆帽额外吐更多三角形。
  5. 最后你拿到的是顶点数组，渲染器只需要按 triangles 去画。
单段线 = 2 个三角形起步 圆角 / 圆帽 = 更多三角形 闭合路径 = 首尾再做一次 joint
9. Checklist

第一次使用 polyline2d，先记住这 8 件事

1. 这不是“画线”，这是“造面”

你拿到的是顶点，不是已经画好的像素线。最后还是要靠你自己的绘制代码去画三角形。

2. 单段线从两个三角形起步

最简单的粗线就是一个四边形，所以最少也要拆成两片三角形。

3. 重复点要小心

连续重复点会产生零长度段，零长度段没有稳定方向和法线。

4. Miter 会变尖刺

角度越尖，外侧交点越远，所以源码会在很尖的情况下自动降级成 bevel。

5. 闭合路径不要再想成两个端点

闭合路径的收尾本质是一个额外拐角，所以要用 JOINT，而不是普通端帽。

6. JOINT 模式下别重复传首尾公共点

库会自己把最后一个点和第一个点接起来，再手动重复往往会让结果变脏。

7. 半透明线条先从 allowOverlap = false 开始

因为重叠三角形可能让局部颜色更深。你先求稳，再去追更激进的轮廓。

8. 圆角和圆帽更好看，但三角更多

视觉柔和通常意味着额外细分。效果和成本之间永远是取舍。

// 一个最小可用的调用套路
