Interactive Geometry Primer

Polyline2D 交互教学

这不是从库名开始的说明书，而是一条更适合新手的路线：先搞懂点、向量、长度、法线、交点、夹角， 再一步一步推到“为什么一条粗折线最后会变成一组三角形”。等这些几何关系都顺了，Polyline2D 的 API 反而会变得很自然。

你会先学到什么

长度、单位向量、法线、点积、叉积、线线相交，这些才是 polyline2d 的地基。

你会后学到什么

斜接、斜切、圆角、端帽、闭合路径、圆弧扇形细分，以及为什么最终输出是三角网格。

这一页怎么读

每节都先讲图和公式，再给少量代码。可以直接拖动滑块，看几何量如何联动。

最终你会得到什么

你会知道 polyline2d 在做什么、为什么这样做、什么时候会退化、什么时候要小心重叠。

一句话先说透

Polyline2D 的核心任务不是“画线”，而是把一串折线点列转成可以交给渲染器的三角形顶点。

输入

按顺序排列的 2D 点。

关键中间量

单位方向、法线、交点、夹角。

输出

一组三角形顶点。

风格控制

MITER、BEVEL、ROUND 与 BUTT、SQUARE、ROUND、JOINT。

先记住这 3 件事

1. 粗线 = 中心线左右各偏移半个宽度。

2. 拐角 = 解决“前一段边界”和“后一段边界”怎么拼。

3. 渲染 = 把结果拆成三角形，因为 GPU 最擅长吃三角形。

01 · 数学铺垫

先从点和向量开始

如果你把 polyline2d 当成“黑盒库”，看到的只是一串点进来、一堆顶点出来；但如果你把它拆开，第一层其实只是高中到大学低年级的二维向量。

向量 AB
单位向量
法线

拖动 A、B 两点的位置，观察原始向量、单位向量和法线如何一起变化。法线就是后面左右偏移的钥匙。

长度与归一化 v = B - A |v| = sqrt(vx^2 + vy^2) u = v / |v|
左法线 如果 u = (ux, uy) 那么 n = (-uy, ux)
点积与叉积 dot(a, b) = ax * bx + ay * by cross(a, b) = ax * by - ay * bx
新手提示

这里最重要的不是记公式，而是记住角色分工：方向决定“往哪走”，法线决定“往哪边扩”。

点列不是面积

输入点列只有一根“骨架线”，本身没有厚度。要画粗线，必须先把骨架扩成带状区域。

单位向量的意义

先把方向长度归一化，才能让“厚度”与线段原始长度无关，不然短线段和长线段的偏移会乱掉。

叉积的意义

在 2D 里，叉积的正负号非常好用。它告诉你第二段是向左转还是向右转，也就是哪一侧是外侧。

02 · 粗线的第一步

把一条中心线扩成一条带

polyline2d 在实现里会先把 thickness 除以 2。原因很直接：我们真正需要的是“半厚度”，这样才能向左偏移一份、向右偏移一份。

中心线
偏移边界
生成的带状面

这是 polyline2d 的最基础模型。单段线没有拐角问题，只需算出左右两条平行边界，再拆成两个三角形。

半厚度 h = w / 2
左右偏移 P+ = P + n * h P- = P - n * h
单段矩形的两个三角形 T1 = (start1, start2, end1) T2 = (end1, start2, end2)
为什么一定是三角形

因为三角形天然共面、稳定、易于光栅化。你可以把任何凸多边形拆成三角形交给 GPU，这是现代图形管线最基本的粒度。

03 · 交点是关键

为什么拐角问题本质上是求交点

当两段中心线拼起来，真正难的不是线本身，而是两段“偏移边界”怎么接。miter 和 bevel 的差别，本质上就是交点怎么取、交点远不远、要不要退化。

线段 A
线段 B
交点

把线段写成参数方程后，交点计算就会非常规整。polyline2d 里就是基于叉积去解 t 和 u。

参数方程 A(t) = P + t * r B(u) = Q + u * s
交点参数 t = cross(Q - P, s) / cross(r, s) u = cross(Q - P, r) / cross(r, s)
平行判断 如果 cross(r, s) 接近 0 说明两条线平行或几乎平行
为什么平行要单独处理

因为分母接近 0 时，交点会非常不稳定。polyline2d 遇到这种情况时会走保守路径，而不是盲目延长出一个很远的尖角。

04 · 真正进入 polyline2d

拐角样式：MITER、BEVEL、ROUND

现在才轮到 polyline2d 的核心观念。相邻线段之间，必须先判断哪边是外侧、哪边是内侧，然后根据样式决定用交点、切角还是圆弧来补那块区域。

中心线
网格填充
三角边
拐角点

把转角拉得很尖时，miter 会变长。polyline2d 内部有一个约 20 度的阈值，太尖就自动退化成 bevel，避免出现夸张的长刺。

角度 theta = arccos( dot(d1, d2) / (|d1| * |d2|) )
外侧判断 cross(d1, d2) < 0 说明顺时针转 cross(d1, d2) > 0 说明逆时针转
miter 尖角长度 miterLength = (w / 2) / sin(theta / 2)
MITER

把两条外边界延长，取交点。最锐利，也最容易在小角度时长得过分。

BEVEL

直接切掉尖角，用一个三角形封口。稳定、便宜，但视觉最硬。

ROUND

用多个小三角形拼一个圆弧。外观柔和，但顶点数量会上去。

05 · 端帽与圆弧

线段两头怎么收尾，圆角又怎么离散成三角形

端帽是粗线的“开头和结尾”。如果路径是开放的，你必须决定端点怎么封。若路径闭合，polyline2d 还支持 JOINT，把尾巴和头再按 joint style 接起来。

中心线
端帽网格

BUTT 不超出端点，SQUARE 会各自向外多延半个厚度，ROUND 会补半圆。

扇形三角片
边界圆弧

polyline2d 内部使用固定最小角度步长来切圆弧。默认常量约等于 10 度，所以圆越大、扫过角越大，需要的三角片就越多。

SQUARE 端帽 start' = start - dir * (w / 2) end' = end + dir * (w / 2)
圆弧扇形三角数 N = max(1, floor(|deltaAngle| / minAngle))
JOINT 端帽的直觉

如果路径闭合，就不再“封口”，而是把最后一段和第一段当成普通拐角来连接。这也是闭环轮廓通常更自然的做法。

06 · 回到库本身

Polyline2D 到底提供了什么功能

有了前面的数学和几何铺垫，再看库就很直白了。它不是场景系统，不是路径编辑器，不是渲染器。它只做一件事：把点列变成粗线网格。

能力 说明 背后的几何动作
输入点列 按顺序传入 2D 点，库会自动跳过完全重合的相邻点，避免零长度线段。 构造中心线段，计算方向和法线。
厚度控制 传入 thickness，内部先除以 2，当作左右偏移量。 用法线把中心线扩成左右两条边界。
关节样式 支持 MITER、BEVEL、ROUND。 延长求交、切角封口、扇形补圆弧。
端帽样式 支持 BUTT、SQUARE、ROUND、JOINT。 平切、外延半厚度、半圆补片、尾首闭合接缝。
闭合路径 选择 JOINT 端帽时，库会自动把最后一点和第一点连接起来，不需要重复传首点。 新增一条连接段，再按普通拐角处理首尾接缝。
allowOverlap 当点距很短而线很粗时，允许一些重叠可获得更完整的覆盖；但透明渲染时可能不理想。 对内侧边界交点的求法更宽松，减少缝隙，但可能引入重叠区域。
输出形式 输出的是顶点序列，本质上按三角形组织，可直接送给图形 API。 每段矩形拆 2 个三角形，圆角和圆帽拆成扇形三角片。
自定义向量类型 只要求点类型有公开的 x、y 浮点成员以及 x、y 构造函数。 模板算法只依赖最基础的坐标读写。
步骤 1

把输入点列切成一个个中心线段，并过滤掉长度为 0 的段。

步骤 2

对每段求单位方向和法线，然后偏移出左右边界。

步骤 3

相邻两段交给 joint 逻辑，根据角度和样式决定怎么拼。

步骤 4

给首尾加端帽，最后把所有带状区域和补片拆成三角形顶点。

实现层面的关键常量

库内部用大约 20 度作为 miter 的最小安全角，用大约 10 度作为圆角三角片的最小步长。这两个常量直接影响尖角退化和圆弧平滑度。

很实用的一点

README 特别强调：普通相邻线段之间不会无意义重绘，只有几何上真的发生重叠时才会出现重复覆盖。这对透明线条很重要。

定位很克制

这是一个 header-only C++17 几何网格生成库，不绑定某个特定渲染后端。你只需要把生成顶点拿去画即可。

08 · 代码示例

              
当你已经理解原理后，再看代码会轻很多

              
下面这些例子都是“短代码 + 清楚意图”。此时你应该能把每行代码背后的几何意义脑补出来，而不是把它们当成不可解释的参数组合。

            

          

          

            

              示例 1：最小可用版本
              
#include <Polyline2D/Polyline2D.h>
using namespace crushedpixel;

std::vector<Vec2> points {
    {-2.0f, -1.0f},
    {-0.5f,  1.2f},
    { 1.4f,  0.2f},
    { 3.0f,  1.8f}
};

auto vertices = Polyline2D::create(
    points,
    0.35f,
    Polyline2D::JointStyle::ROUND,
    Polyline2D::EndCapStyle::ROUND
);

// 把 vertices 当三角形顶点渲染

              
这就是最典型的用法：输入点列、厚度、拐角样式、端帽样式，得到的就是可渲染顶点。

            

            

              示例 2：闭合轮廓
              
std::vector<Vec2> polygon {
    {-1.5f, -1.2f},
    { 0.0f,  1.8f},
    { 2.0f,  0.6f},
    { 1.1f, -1.6f}
};

auto vertices = Polyline2D::create(
    polygon,
    0.25f,
    Polyline2D::JointStyle::MITER,
    Polyline2D::EndCapStyle::JOINT
);

// 注意：不要再手动把首点重复追加到末尾

              
JOINT 端帽表示“尾巴和头接起来”。库会自动生成最后一段到第一点的连接段。

            

            

              示例 3：透明线条时的取舍
              
std::vector<Vec2> veryDense {
    {-2.0f, 0.0f},
    {-1.5f, 0.1f},
    {-1.0f, 0.0f},
    {-0.6f, 0.2f},
    {-0.2f, 0.0f}
};

auto vertices = Polyline2D::create(
    veryDense,
    0.8f,
    Polyline2D::JointStyle::BEVEL,
    Polyline2D::EndCapStyle::BUTT,
    false
);

// allowOverlap=false 更适合透明渲染

              
点间距比厚度还小时，allowOverlap=true 可能让外形更饱满；但如果要做透明混合，最好先保守一些。

            

            

              示例 4：自定义点类型
              
struct MyPoint {
    float x;
    float y;

    MyPoint(float xValue, float yValue)
        : x(xValue), y(yValue) {}
};

std::vector<MyPoint> points {
    {-1.0f, -1.0f},
    { 0.0f,  1.0f},
    { 2.0f,  0.2f}
};

auto vertices = Polyline2D::create(
    points,
    0.4f,
    Polyline2D::JointStyle::MITER,
    Polyline2D::EndCapStyle::SQUARE
);

              
库不强行绑定自己的 Vec2，只要求你的点类型暴露 x、y 并且能用两个浮点数构造。

            

          

        


        

          

            

              
09 · 常见误区

              
哪些地方最容易把新手绕进去

              
如果你已经看到这里，最后值得做的是把几个常见误区捋清，不然实际使用时会觉得“结果不对”，但其实只是几何直觉还没对上。

            

          

          

            

              误区 1：thickness 是向一边偏移的距离
              
不是。thickness 是总宽度，库内部会改成一半，左右各扩一份。

            

            

              误区 2：miter 一定比 bevel 更好看
              
不一定。角度太尖时，miter 会长得非常夸张，所以库内部会自动退回 bevel。

            

            

              误区 3：ROUND 只是换个开关
              
不是。ROUND 背后意味着额外三角片，顶点数和性能开销都会增加。

            

            

              误区 4：闭合路径要重复首点
              
如果你用的是 JOINT 端帽，不需要重复首点。重复了反而可能多出一段零长度或重复接缝。

            

            

              误区 5：透明线条与不透明线条处理完全一样
              
allowOverlap 的策略就提醒你：透明混合时，重叠区域会被画多次，视觉上可能发黑或发亮。

            

            

              误区 6：输出顶点还是“线”
              
不是。输出已经是填充三角形顶点，你要用三角形方式提交，而不是再把它当线段去画。

            

          

          

            如果只想快速背下整个算法，应该记什么？
            

              
记这条链就够了：点列 -> 线段方向 -> 单位法线 -> 左右偏移边界 -> 相邻段交点/切角/圆弧 -> 首尾端帽 -> 三角形顶点流。

            

          

          

            为什么 polyline2d 不直接输出“矩形列表 + 圆弧参数”？
            

              
因为最终接到图形 API 时，最通用、最稳定的数据就是顶点。越早把几何离散掉，后续渲染环节越简单。

            

          

          

            读到这里，建议你再回头看一遍“拐角样式”和“全流程实验室”。如果你能仅凭图示解释出为什么某个尖角会退成 bevel，为什么圆角需要若干个扇形三角片，那就说明你已经真的理解 polyline2d 了。
