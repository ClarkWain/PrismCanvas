2. 预备知识：几行高中数学

整个 Polyline2D 只需要 三个向量运算，Vec2.h 全实现了：

Vec2.h — 向量基础
struct Vec2 {
  float x, y;
  Vec2(float x, float y) : x(x), y(y) {}
};

// 加减乘除
Vec2 add(Vec2 a, Vec2 b)     { return {a.x+b.x, a.y+b.y}; }
Vec2 sub(Vec2 a, Vec2 b)     { return {a.x-b.x, a.y-b.y}; }
Vec2 mul(Vec2 v, float s)  { return {v.x*s, v.y*s}; }

// 长度 & 单位向量
float mag(Vec2 v)            { return sqrt(v.x*v.x + v.y*v.y); }
Vec2 norm(Vec2 v)            { return mul(v, 1/mag(v)); }

// 点积 & 叉积
float dot(Vec2 a, Vec2 b)   { return a.x*b.x + a.y*b.y; }
float cross(Vec2 a, Vec2 b) { return a.x*b.y - a.y*b.x; }

2.1 法向量 — 整个库的支点

dir = (dx, dy)  →  normal = (-dy, dx)
LineSegment.h — 法向量计算
Vec2 normal() const {
  auto dir = direction();    // 单位方向向量
  return {-dir.y, dir.x};    // 逆时针旋转 90°
}

💡 为什么法向量这么重要？

"给线段加粗" = 把端点沿法向量方向各推开半个线宽。

线段有一个方向（dir），垂直于方向的就是法向量（normal）。把 P₀ 沿 normal 方向移出 t/2，沿反方向移出 t/2，就得到了加粗矩形的两个角点。

原始线段 法向量方向 (normal) 偏移后的轮廓

3. 核心操作：把线段"撑"成矩形

这是 Polyline2D 里 PolySegment 做的事情：

Polyline2D.h — PolySegment 结构体（核心）
template<typename Vec2>
struct PolySegment {
  PolySegment(const LineSegment<Vec2> ¢er, float thickness) :
    center(center),
    // 把中心线沿法向量方向平移 thickness
    edge1(center + center.normal() * thickness),  // ← 左轮廓
    edge2(center - center.normal() * thickness)   // ← 右轮廓
  {}

  LineSegment<Vec2> center;  // 原始线段
  LineSegment<Vec2> edge1;   // 左边缘
  LineSegment<Vec2> edge2;   // 右边缘
};

等等，为什么是 + 和 -，而不是 + 和 +？ 因为 `normal()` 返回的是单位法向量，加一次得到一侧，减一次得到另一侧：

edge1 = center + normal × thickness     edge2 = center − normal × thickness

3.1 从四条边到两个三角形

有了 edge1 (a→b) 和 edge2 (a→b)，我们有了矩形的四个顶点。只需要两个三角形就能覆盖它：

create() 中的三角形生成（核心生成逻辑）
// 对每一段（segment），生成两个三角形 = 6个顶点
*vertices++ = edge1.a;     // 左上
*vertices++ = edge2.a;     // 左下
*vertices++ = edge1.b;     // 右上

*vertices++ = edge1.b;     // 右上
*vertices++ = edge2.a;     // 左下
*vertices++ = edge2.b;     // 右下

🔑 关键

每段生成 6 个顶点（2 个三角形），使用 GL_TRIANGLES 模式渲染。每段的三角形是独立的，不支持三角形带复用。

4. 拐角处理

两条线段衔接处，如果各画各的矩形，会出现一个 缺口：

Polyline2D 用 createJoint() 处理这个问题。它有 三种拐角样式：

4.1 MITER — 斜接

createJoint() — MITER 实现
if (jointStyle == JointStyle::MITER) {
  // 计算相邻两条外侧边界的交点 (作为无限直线求交)
  auto sec1 = LineSegment::intersection(
    segment1.edge1, segment2.edge1, true);
  auto sec2 = LineSegment::intersection(
    segment1.edge2, segment2.edge2, true);

  // 用交点替换原来的端点
  end1 = sec1 ? *sec1 : segment1.edge1.b;
  end2 = sec2 ? *sec2 : segment1.edge2.b;

  // 下一段的起点延续同样的交点
  nextStart1 = end1;
  nextStart2 = end2;
}

思路：把两条外侧边延长，让它们相交于一点。用这个交点代替原来的断点，缺口就消失了。

两条直线 A + r·t 和 B + s·u 相交 ↦ 解 t 和 u

⚠️ 夹角太小会出问题

当两条线段夹角很小时，交点会飞到极远处，产生长长的"尖刺"。这需要 Miter Limit 来处理（详见第 7 节）。

4.2 BEVEL — 斜角

createJoint() — BEVEL 实现
if (jointStyle == JointStyle::BEVEL) {
  // 三个顶点形成一个填充三角形：
  // 外侧边缘1终点 + 外侧边缘2起点 + 内侧交点
  *vertices++ = outer1->b;
  *vertices++ = outer2->a;
  *vertices++ = innerSec;
}

思路：不追求无缝斜接，而是在缺口处直接画一个三角形填上。Bevel = 斜削掉尖角。

4.3 ROUND — 圆角

createJoint() — ROUND 实现（核心：三角扇）
if (jointStyle == JointStyle::ROUND) {
  // 在拐角处画一圈小三角形模拟圆弧
  createTriangleFan(vertices, innerSec, segment1.center.b,
                    outer1->b, outer2->a, clockwise);
}

// --- createTriangleFan 的原理 ---
// 计算 start→origin 和 end→origin 之间的角度差
float angle1 = atan2(point1.y, point1.x);
float angle2 = atan2(point2.y, point2.x);

// 分成 N 个小三角形，每个约 10°
int numTriangles = floor(abs(jointAngle) / roundMinAngle);
float triAngle = jointAngle / numTriangles;

// 逐三角发射
for (int t = 0; t < numTriangles; t++) {
  // 旋转 point1 得到当前顶点
  endPoint.x = cos(rot) * point1.x - sin(rot) * point1.y;
  endPoint.y = sin(rot) * point1.x + cos(rot) * point1.y;
  endPoint += origin;  // 平移回原本位置

  *vertices++ = startPoint;
  *vertices++ = endPoint;
  *vertices++ = connectTo;  // 内侧交点（三角形扇的公共顶点）
  startPoint = endPoint;
}

createTriangleFan 是整个库最巧妙的函数。只需指定中心点、起点和终点，它自动把圆弧切成小三角形。分段密度由 roundMinAngle ≈ 10° 控制：

分段数 = ⌊|夹角| / 10°⌋  →  每个三角形 ≈ 10°

5. 内侧 vs 外侧的判定

拐角处，哪边是"内"、哪边是"外"？这决定了哪些边需要求交、哪些边可以直接连。

Polyline2D 用 叉积 判断转向：

判定顺时针/逆时针
auto dir1 = segment1.center.direction();
auto dir2 = segment2.center.direction();

// 叉积的 z 分量决定转向
auto clockwise = dir1.x * dir2.y - dir1.y * dir2.x < 0;
// 用 cross = a.x*b.y - a.y*b.x

法向量的约定：Polyline2D 定义法向量 = (-dir.y, dir.x)，即逆时针旋转 90°。所以：

转向左侧 (edge1)右侧 (edge2)
顺时针 (clockwise)OUTER（外）INNER（内）
逆时针 (ccw)INNER（内）OUTER（外）
createJoint() 内外判定
if (clockwise) {
  outer1 = &segment1.edge1;  // 左 = 外
  outer2 = &segment2.edge1;
  inner1 = &segment1.edge2;  // 右 = 内
  inner2 = &segment2.edge2;
} else {
  outer1 = &segment1.edge2;  // 右 = 外
  outer2 = &segment2.edge2;
  inner1 = &segment1.edge1;  // 左 = 内
  inner2 = &segment2.edge1;
}

🎯 为什么区分内外很重要？

外侧：MITER 时外侧边延长求交。BEVEL/ROUND 时外侧边端点之间需要填充三角形。

内侧：内侧边求交作为"内侧顶点"，所有拐角的填充三角形都以此为公共顶点。

6. 端头处理（End Cap）

路径的起点和终点也需要处理，否则是垂直的切面。

端头处理的四种方式
if (endCapStyle == EndCapStyle::SQUARE) {
  // 往外推半个线宽
  pathStart1 -= edge1.direction() * thickness;
  pathStart2 -= edge2.direction() * thickness;
  pathEnd1   += edge1.direction() * thickness;
  pathEnd2   += edge2.direction() * thickness;

} else if (endCapStyle == EndCapStyle::ROUND) {
  // 画半圆三角扇
  createTriangleFan(vertices, firstSegment.center.a,
    firstSegment.center.a,
    firstSegment.edge1.a, firstSegment.edge2.a, false);
  createTriangleFan(vertices, lastSegment.center.b,
    lastSegment.center.b,
    lastSegment.edge1.b, lastSegment.edge2.b, true);

} else if (endCapStyle == EndCapStyle::JOINT) {
  // 闭合路径：用拐角处理连接首尾
  createJoint(vertices, lastSegment, firstSegment, ...);
}
// BUTT：什么都不做，直接是垂直切面

BUTT：什么都不做，端面垂直于路径方向。

SQUARE：把端头外推半个线宽，产生"方头"效果。

ROUND：在端头画半圆（用 createTriangleFan）。

JOINT：用于闭合路径时，把尾和首用拐角逻辑连接。

7. Miter Limit：防止尖刺

当两条线夹角很小时，外侧边的交点会飞出去极远，产生难看的尖刺：

Miter Limit 实现
constexpr float miterMinAngle = 0.349066f;  // ≈ 20°

// 计算两个方向向量的夹角
float angle = Vec2Maths::angle(dir1, dir2);
float wrappedAngle = angle;
if (wrappedAngle > pi / 2) {
  wrappedAngle = pi - wrappedAngle;  // 取锐角
}

if (wrappedAngle < miterMinAngle) {
  // 夹角太小 → 自动降级为 BEVEL
  jointStyle = JointStyle::BEVEL;
}

为什么是 20°？ 这是经验值。过小会产生长尖刺，过大则太多 miter 被降级。Skia 和 SVG 默认使用约 4.0 的 miter limit（对应 ~28.96°）。

8. 三角形输出：送给 GPU

Polyline2D 输出的是 连续的三角形顶点流，每个三角形 3 个顶点，每段 2 个三角形，拐角和端头额外增加。

9. 虚线：动态切分路径

PrismCanvas 在调用 Polyline2D 之前做了虚线拆分。这不是 Polyline2D 的一部分，但了解它对理解完整管线有帮助：

Canvas.cpp — 虚线拆分逻辑
// 沿路径前进，按 dash pattern 切分
float remaining = intervals[intervalIndex] - phase;
bool drawing = (intervalIndex % 2) == 0;  // 偶数为画，奇数为空
std::vector<Vec2> currentDash;

for (auto segment : segments) {
  while (consumed < length) {
    float step = min(remaining, length - consumed);
    auto next = pointOnSegment(start, end, t);

    if (drawing) currentDash.push_back(next);

    remaining -= step;
    if (remaining <= epsilon) {
      if (drawing) finishDash();  // 一段虚线完成
      drawing = !drawing;             // 切换画/空
      remaining = nextInterval;
    }
  }
}

// 每段虚线再调用 buildStrokeMesh → Polyline2D::create
for (auto &dash : dashes) {
  submitStrokeMesh(renderer, dash, false, dashPaint, ...);
}

📐 为什么叫 Polyline2D？

它处理的不是"曲线"而是"折线"（polyline = 多段线）。如果需要圆滑曲线（如贝塞尔曲线），调用方需要事先把曲线展平（flatten）成一系列短直线段。

PrismCanvas 的 Path 类中 quadTo 和 cubicTo 做的就是这件事：把曲线切成 8~96 段小直线，然后再送入 Polyline2D。
